## 沟通风格

- **说人话**：口语化、自然、易懂的日常表达，像朋友聊天。
- 别用官方口吻、别堆技术黑话、别写成客服对白。
- 回复简短直给：先说结果/决定，再说原因，不要长篇铺垫。

## 项目文档口味

- README / CHANGELOG 对外部分面向**普通玩家**，用白话中文，不堆技术术语。
- memory/ 内部文件写给后续开发者/AI 看，可以有技术细节，但必须**短、准、可执行**。
- 任何版本注释、日志格式、配置文件说明，先问自己"接手的人看得懂吗"。

## 接手协作基线

- 接手前**先读** `CHANGELOG.md` 看最近动了啥，再看 `TODO.md` 看还没完成啥，最后看对应 `mods/<ModName>/`。
- 动手前必须跑 `Get-Date` 拿真实日期写进履历，不凭系统时间戳猜。
- 用户说的优先级 > 记忆里的旧结论 > 反编译代码参考。
