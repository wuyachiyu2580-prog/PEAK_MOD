# 协作与记忆规则

## 需求澄清铁律

- 不允许把用户没有明确要求删除、停用或重构的功能按猜想处理。
- 如果需求存在歧义，尤其涉及“删除代码”“停用 UI”“改变联机同步/广播协议”“移除诊断或 HUD”时，必须先向用户提出具体澄清问题，再动手。
- 禁止用“我以为”“看起来不需要”“顺手清理”作为修改理由；不确定就保留现有功能边界。

更新时间：2026-05-19

## memory 文件夹定位

`memory/` 是给**其他软件/协作方快速上手项目**用的公开知识库，必须实时、权威、一致。任何动作前后都要维护这个特性。

## 读取规则（任务开始前必做）

1. **list_dir 查目录**：看 `memory/` 下各文件的**大小与修改时间**，判断是否有人动过。
2. **翻 CHANGELOG.md**：有变化时先读最新条目，一句话速览近期改了啥。
3. **读根 README.md 和 MEMORY_INDEX.md**：确认当前结构与读取顺序。
4. **按需 read_file**：处理具体 MOD 时，必须读 `mods/<ModName>/` 下的 README / RECENT / DECISIONS / FILES。
5. **读最近临时记忆**：处理具体 MOD 时，先看 `mods/<ModName>/temp/` 下最新的 `YYYY-MM-DD.md`，尤其是上下文压缩、会话中断或换 AI 之后。
6. **通用规则**：只在需要跨 MOD 规范时读 `common/*.md`。
7. **待办与风险**：看根 `TODO.md`。

> 不再依赖已删除的历史文件；缺失信息必须从**源码、日志或用户反馈**重新确认。

## 写入规则（新增/修改记忆时，四步缺一不可）

用户反复确认的**四同步铁律**：

1. **`update_memory`**：按 category 创建/更新/删除内存记忆实体。
2. **改对应 MD 文件**：
   - MOD 专属信息 → `mods/<ModName>/` 下的 README / RECENT / DECISIONS / FILES。
   - 跨 MOD 通用规则 → `common/`。
   - 永久待办、未完成、风险 → 根 `TODO.md`。
   - 新主题就新开文件，不要挤进不相关的文件。
3. **更新 `MEMORY_INDEX.md`**：把新文件/新章节加进条目清单，保持索引与实际文件一一对应。
4. **追加 `CHANGELOG.md`**：在顶部新日期块里加一条履历，格式：
   ```
   - [新增/修改/删除/规则/索引] 文件名：一句话说清楚改了啥
   ```

> **一致性铁律**：内存记忆库 ↔ 本地 MD 文件库 ↔ MEMORY_INDEX.md ↔ CHANGELOG.md，**四者严格一致**。
> 别只改记忆不改 MD，也别漏记履历。

## 临时思考记忆规则

- 每个 MOD 必须有自己的临时区：`mods/<ModName>/temp/`。
- 当前日期的临时文件命名为 `YYYY-MM-DD.md`，日期仍按 `Get-Date -Format "yyyy-MM-dd"` 获取。
- 每处理某个 MOD 时，每形成 3 次明确的阶段性判断、排查结论或方案取舍，就把**思考结果摘要**追加到当天临时 MD。
- 临时 MD 只写压缩后有用的信息：当前判断、证据、风险、下一步；不要写原始脑内推理链，也不要把无关流水账塞进去。
- 上下文压缩、会话中断、换 AI 或长时间回到任务时，必须先读该 MOD `temp/` 下最近生成的日期文件，再继续读正式 `RECENT.md` / `DECISIONS.md`。
- 临时结论被验证稳定后，应再整理进正式四件套、根 `TODO.md` 或 `CHANGELOG.md`；临时 MD 不能替代正式记忆。

## 日期规则

取当天真实日期的优先级：

1. **首选**：PowerShell 下跑 `Get-Date -Format "yyyy-MM-dd"`（或带星期的 `"yyyy-MM-dd dddd"`），命令返回什么就用什么。
2. **兜底**：命令执行失败、跨时区异常或取不到时，才问用户"今天几月几日？"。

**禁止**：
- 凭系统提示的时间戳猜。
- 凭对话历史或印象套用。
- 写成 "4 月 26 日" 这种错误（用户明确强调过：系统时间是那天，真实日期是 5 月 5 日）。

## 内容质量

- 写给后续接手者，不写个人流水账。
- 每个结论必须说明**当前状态、风险或下一步**。
- 禁止把已经废弃的方案写成当前方案。
- 禁止瞎编——**不确定就写 "待补充" 或 "待从源码确认"**，不要编造代码/版本号/文件名。
